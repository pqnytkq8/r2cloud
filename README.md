# 利用Cloudflare R2 + Workers搭建在线网盘


[汉化修改自/longern/FlareDrive](https://github.com/longern/FlareDrive)

增加了权限系统，支持多管理员，分别授权目录

cloudflare R2是一个文件储存系统，配合Cloudflare Workers可以实现这样一个网盘系统

### 搭建教程


1. fork该仓库
2. 前往Cloudflare R2新建一个R2储存桶
   - **【推荐】不允许公开访问** - 更安全，所有文件通过 Workers 代理访问，无法直链盗刷
   - 或选择"允许公开访问"（较低安全级别）
3. 前往Cloudflare Pages新建一个站点，选择连接到Git

4.选择刚刚fork的仓库，点击开始设置
5.项目名称可以修改，其他项目保持默认不动

6.展开环境变量，添加

| 变量名称| 值| 说明 |
| --- | --- | --- |
| PUBURL | ~~复制的**公共存储桶URL**~~ | **【已弃用】新版本无需配置，自动通过 Workers 代理** |
| GUEST | public/ | 游客允许读取/写入的目录，逗号分隔 |
| admin:123456 | * | 账户格式：`账号:密码`，值为允许的目录（`*` 表示全部） |
| user1:123456 | user1/,userPublic/ | 支持多个用户，各自不同权限 |

以此类推，GUEST 代表游客的允许读写目录

管理员则以`账号:密码`的形式设置，值代表其允许读写的目录，用`,`隔开，**请勿在前后加逗号，否则会授予所有目录的读写权限**

设置好后点击**开始部署**

7.前往Pages->cloudflare-r2-oss->设置->函数->R2 存储桶绑定,绑定R2存储桶,变量名称`BUCKET`

8.在部署页面重新部署即可

### 安全说明

**【重要】存储桶配置**
- 所有文件访问都通过 Cloudflare Workers 代理，支持权限验证
- 建议将 R2 存储桶设置为**私有**（不允许公开访问）
- 即使设置为公开，也无法直接通过公开 URL 访问，必须通过应用认证

**【优势】与公开方式的对比**
| 特性 | 私有存储桶（推荐） | 公开存储桶 |
|------|---------|---------|
| 直链盗刷风险 | ❌ 无 | ⚠️ 高 |
| 权限控制 | ✅ 完整 | ⚠️ 仅应用层 |
| 带宽费用 | ✅ 最小化 | ⚠️ 容易超支 |
| 缓存效率 | ✅ 完整 | ✅ 完整 |
| 文件分享 | ✅ 支持临时链接 | ✅ 支持 |
